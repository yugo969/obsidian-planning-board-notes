# Planning Board Notes

Obsidian の Markdown ノートをデータ源にして、プランニングボードを表示するローカルプラグインです。

## インストール

スマホでは BRAT 経由でこの repo を追加して使います。

1. Obsidian の Community plugins で BRAT をインストールして有効化します。
2. BRAT の `Add Beta plugin` で、この GitHub repo のURLを追加します。
3. Community plugins に `Planning Board Notes` が出たら有効化します。

## データ

- グループノート: `Planning Board/Groups/*.md`
- タスクノート: `Planning Board/Tasks/*.md`
- タスクの状態、期限、種別、アーカイブ状態は frontmatter に保存します。
- タスク本文には、詳細メモ、参照、元の依頼を Markdown として残します。

## 使い方

1. Obsidian を再起動するか、Community plugins で `Planning Board Notes` を有効化します。
2. 左リボンの dashboard アイコン、またはコマンドパレットの `Open Planning Board Notes` を開きます。
3. 上部の `期限カード` / `月表示` / `週表示` で、カテゴリ中心・月間カレンダー・週単位の見方を切り替えます。
4. `本日` エリアでは、今日の日付、本日期限の件数、期限超過、次の期限を確認できます。
5. ボード内で状態セレクトを変えると、該当タスクノートの `status` が更新されます。
6. 通常表示では、未アーカイブのタスクだけをカテゴリごとに表示します。
7. `アーカイブ表示` に切り替えると、アーカイブ済みタスクを持つカテゴリだけを表示します。
   一部だけアーカイブされているカテゴリは、通常表示とアーカイブ表示の両方に出ます。
8. `タスクをアーカイブ` / `アーカイブ解除` は、対象タスクノートの `archived` を更新します。
9. `全体をアーカイブ` / `全体を解除` は、カテゴリノートの `archived` を更新します。
   全体アーカイブ中のカテゴリは通常表示から消え、アーカイブ表示では全タスクを表示します。
10. `タスク非表示` / `タスク表示` は、カテゴリ内のタスクカード領域だけを折りたたみます。
11. 狭い表示幅では、カテゴリ見出しが画面上部に固定され、カテゴリタイトルは見出し内で横スクロールできます。判定は端末種別ではなくボード表示領域の幅で行います。
12. `月表示` は狭い表示幅でも7列のカレンダーを維持します。カレンダー内のタスクは短く表示し、タップまたはクリックでカテゴリとタスクカードをモーダル確認できます。
13. 月表示のモーダル内でも完了チェックを更新できます。`カテゴリの他タスクを見る` で同じカテゴリの他タスクを確認できます。
14. `詳細` はノート本文をモーダル表示し、`ノート` は元ノートを開きます。
