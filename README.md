# Planning Board Notes

Obsidian の Markdown ノートをデータ源にして、プランニングボードを表示するローカルプラグインです。

## インストール

スマホでは BRAT 経由でこの repo を追加して使います。

1. Obsidian の Community plugins で BRAT をインストールして有効化します。
2. BRAT の `Add Beta plugin` で、この GitHub repo のURLを追加します。
3. Community plugins に `Planning Board Notes` が出たら有効化します。

## データ

- グループノート: `Planning Board/Groups/*.md`
- タスクノート: `Planning Board/Tasks/*.md`
- タスクの状態、期限、種別、アーカイブ状態は frontmatter に保存します。
- タスク本文には、概要、参照、必要なメモを Markdown として残します。
- タスク frontmatter に `memo_label` がある場合、その見出しと一致する本文セクションだけをカード内のメモボタンから開けます。

## 使い方

1. Obsidian を再起動するか、Community plugins で `Planning Board Notes` を有効化します。
2. 左リボンの dashboard アイコン、またはコマンドパレットの `Open Planning Board Notes` を開きます。
3. 上部の `期限カード` / `月表示` / `週表示` で、カテゴリ中心・月間カレンダー・週単位の見方を切り替えます。
4. `本日` エリアでは、今日の日付、本日期限の件数、期限超過、次の期限を確認できます。
5. ボード内で状態セレクトを変えると、該当タスクノートの `status` が更新されます。
6. 通常表示では、未アーカイブのタスクだけをカテゴリごとに表示します。
7. `アーカイブ表示` に切り替えると、アーカイブ済みタスクを持つカテゴリだけを表示します。
   一部だけアーカイブされているカテゴリは、通常表示とアーカイブ表示の両方に出ます。
8. `タスクをアーカイブ` / `アーカイブ解除` は、対象タスクノートの `archived` を更新します。
9. `全体をアーカイブ` / `全体を解除` は、カテゴリノートの `archived` を更新します。
   全体アーカイブ中のカテゴリは通常表示から消え、アーカイブ表示では全タスクを表示します。
10. `タスク非表示` / `タスク表示` は、カテゴリ内のタスクカード領域だけを折りたたみます。
11. タスク表示切替、完了チェック、アーカイブ選択、フィルタ変更などの再描画操作では、現在のスクロール位置を維持します。
12. 狭い表示幅では、カテゴリ見出しが画面上部に固定され、カテゴリタイトルは見出し内で横スクロールできます。判定は端末種別ではなくボード表示領域の幅で行います。
13. `月表示` は狭い表示幅でも7列のカレンダーを維持します。カレンダー内のタスクは短く表示し、タップまたはクリックでカテゴリとタスクカードをモーダル確認できます。
14. 月表示のモーダル内でも完了チェックを更新できます。チェック操作は完了状態だけを変え、表示中のタスクは切り替えません。`カテゴリの他タスクを見る` から控えめな `このタスクを見る` リンクを押すと表示タスクを切り替え、`最初のタスクに戻る` リンクでカレンダーから開いたタスクへ戻れます。
15. タスクカード内の `詳細` は概要と元会話をその場で表示し、タスクに `memo_label` がある場合は `清掃メモ` や `当日メモ` などのメモボタンを表示します。メモボタンはノート全体ではなく、該当する本文セクションだけをモーダル表示します。月表示で開いたタスクモーダル内でも同じメモボタンが使えます。`ノート` は元ノートを開きます。月表示のタスクモーダル内では、ノートを見える状態で開くためにタスクモーダルを閉じてから遷移します。カテゴリの `候補メモ` や `時期メモ` も、そのメモ本文だけをモーダル表示します。
16. 通常表示でフィルタやアーカイブ表示を使っていないときは、カテゴリ見出しやタスクカードの操作部品以外の背景をドラッグして並び替えできます。保存した順序は同期APIに保存され、Web版とObsidian plugin版で同じ順序を読みます。
17. `同期コード` から Web 版の進捗同期と同じコードを保存すると、カテゴリ開閉、タスク詳細の開閉状態、カテゴリ/タスクの表示順を同期APIへ保存します。タスク詳細は初期状態では閉じ、明示的に開いたタスクだけを同期し、閉じると同期対象から外します。完了状態やアーカイブ状態はタスクデータなので、ObsidianノートのfrontmatterとWeb版からの同期スクリプトで扱います。
